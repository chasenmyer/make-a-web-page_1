Lesson 1: The Basics of the Web and HTML

How the Web Works

The web is a collection of HTML documents. You use your computer to go to a web page like www.google.com, the computer sends a HTTP Request to a server which finds the appropriate HTML document and sends it back to your computer where a web browser interprets the page and displays it on the user's screen. Steve Huffman draws it out here.

HTML: Hypertext Markup Language

HTML docs are the majority of the content on the web. HTML documents contain text content (what you see) and markup (how it looks). Steve explains it quickly here

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. All the pieces explained here

Inline vs Block Elements

HTML elements are either inline (continue in a line) or block (form an "invisible box" around the content)

DOM: Document Object Model

a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

The relationship between indented HTML and boxes

An HTML document in text format with good legible syntax will show a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested the "box" is.

Text Editors (for programming)

Notepad can be used but editors like sublime help to guess what you will use, has a color coding to read the text easier and auto generates the html tag.

Lesson 3: Adding CSS Style and HTML Structure

CSS (Cascading Style Sheets)

Lets you recycle style code without having to retype it with the most specific rule applying to every element

You can also think about CSS as a search and replace tool: you identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones.

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

Adding Style: Jessica describes it in this video

Selector defines what elements a style will apply to: select a tag and it will apply to all of those tags on a page. It can also be applied to a class
